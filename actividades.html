Propósito del Portafolio

El propósito de este portafolio de evidencias es documentar y demostrar el proceso de aprendizaje del estudiante a lo largo del desarrollo de la asignatura Desarrollo Web Orientado a Servicios. En él se integran actividades, prácticas y proyectos que evidencian la adquisición de conocimientos teóricos y habilidades prácticas en el diseño y construcción de aplicaciones web basadas en servicios. Este portafolio permite evaluar el progreso académico, fortalecer el aprendizaje autónomo y servir como referencia del trabajo realizado durante el curso.

¿Qué aprenderá el usuario?

El usuario (estudiante) aprenderá a: Comprender el paradigma del desarrollo orientado a servicios Identificar los servicios que se ofrecen en la nube Analizar la Arquitectura Orientada a Servicios (SOA) Reconocer y aplicar estándares como: XML SOAP WSDL UDDI REST Diseñar aplicaciones web que consuman servicios Integrar contenidos mediante aplicaciones web híbridas (Mashup) Organizar información y evidencias de forma estructurada en un entorno web

Objetivo del Proyecto

El objetivo del proyecto es desarrollar un portafolio de evidencias digital que integre de manera ordenada y visual las actividades realizadas durante el curso, aplicando los conceptos del desarrollo web orientado a servicios. Este proyecto busca reforzar el aprendizaje práctico, promover el uso de tecnologías web y demostrar la correcta implementación de principios, estándares y arquitecturas de servicios.

Competencias Desarrolladas

  • Diseño de arquitecturas orientadas a servicios (SOA)
  • Consumo e implementación de APIs
  • Integración de servicios web
  • Diseño de interfaces web modernas

Resultados Esperados

Al finalizar el proyecto se espera que el estudiante: Comprenda y aplique el enfoque orientado a servicios Diseñe un portafolio web funcional y estructurado Integre correctamente información teórica y práctica Presente evidencias claras del aprendizaje adquirido Utilice buenas prácticas en el desarrollo web Demuestre dominio básico-intermedio en tecnologías web

Tecnologías Utilizadas

  • HTML5 y CSS3
  • JavaScript
  • APIs REST
  • Arquitectura SOA

Metodología

La metodología utilizada en el desarrollo del proyecto es teórico-práctica, basada en: Investigación documental de conceptos y estándares Análisis de casos y ejemplos reales Desarrollo de actividades prácticas por unidad Organización de evidencias en un entorno web Uso de tecnologías web como HTML, CSS, JavaScript y servicios web Evaluación continua mediante la revisión de actividades y resultados Esta metodología permite un aprendizaje progresivo, práctico y enfocado en la aplicación real de los conocimientos.

Unidad 1

Introducción a SOA y servicios web.

En esta unidad se estudian los fundamentos del desarrollo web orientado a servicios, identificando el paradigma SOA, los servicios en la nube, las aplicaciones web híbridas (Mashup) y los principales estándares utilizados para la comunicación entre servicios.

Ver Unidad 1

Unidad 2

Diseño y consumo de servicios REST.

Esta unidad aborda el diseño e implementación de servicios web REST, el consumo de APIs, el manejo de peticiones HTTP y la aplicación de buenas prácticas para el desarrollo de servicios eficientes y escalables.

Ver Unidad 2

Unidad 3

Integración de APIs y buenas prácticas.

Se enfoca en la integración de múltiples APIs dentro de aplicaciones web, considerando aspectos de seguridad, autenticación, manejo de datos y optimización del rendimiento.

Ver Unidad 3

Unidad 4

Proyecto final y evidencias.

En esta unidad se desarrolla el proyecto final integrador, donde se aplican los conocimientos adquiridos durante el curso y se presentan las evidencias del trabajo realizado.

Ver Unidad 4
